# Paraphrase test: is the performance model just copying the memory model?

**Rubric 7d.** *Take ~30 cards. For each, write 2+ exam-style questions that test
the same idea in new words. Compare the student's recall on the card with their
accuracy on the reworded questions. If the two numbers are basically the same,
the performance model is just copying the memory model. Report the gap.*

**Headline result (from the run):**

| Signal | Mean |
|---|:---:|
| Memory recall on the *original* card | **86.8%** |
| Performance on the *reworded* paraphrases | **71.3%** |
| **Gap (paraphrase − memory)** | **−0.155 (−15.5 pts)** |

The gap is **clearly nonzero and negative**: paraphrases (same idea, new wording)
are harder than raw recall of the memorized card. So the performance signal is
**not** an echo of the memory signal — it captures something recall alone misses.

> **This is a clearly-labelled SIMULATION, not real students.** There were no
> real learners to sit the paraphrase test this week. Per rubric §9, a labelled
> simulation beats a fake number and honesty scores highest — so both numbers
> above are **derived from the app's own Rasch/1PL model**, not hand-picked. See
> [The honest caveat](#the-honest-caveat).

---

## The corpus (real, on disk)

The reworded questions are **not** generated on the fly — they were authored and
committed to [`content/items.json`](../content/items.json). Each source card
carries a `paraphrases` array; each paraphrase is a full exam-style item (stem,
choices, answer, explanation, `parent_id`) that tests the *same idea in new
words*.

- **29 source cards** each have **≥ 2** linked paraphrases (rubric asks for ~30, 2 each).
- **83 paraphrase questions** in total across those 29 cards.
- Coverage spans Quant (Arithmetic, Algebra, Word problems) and Verbal (Critical
  Reasoning, Reading Comprehension).

Example (`Q-PS-001`, "Arithmetic: percents") — the source card and one of its
paraphrases test the identical idea (successive ± percent changes don't cancel)
in different words:

> **Paraphrase `Q-PS-001-p1`:** *"A price is increased by 10% and then the new
> price is decreased by 10%. The final price is what percent of the original?"*
> → B) 99% (1.10 × 0.90 = 0.99).

The experiment set is discovered programmatically: any item in `items.json` with
`len(paraphrases) >= 2` is included (see `experiment_set()` in
[`content/scripts/memory_vs_performance.py`](../content/scripts/memory_vs_performance.py)).

## The metric

For each source card, per
[`content/scripts/memory_vs_performance.py`](../content/scripts/memory_vs_performance.py):

```
memory_recall = P(recall the EXACT, memorized card)          # the familiar wording
perf_accuracy = fraction of that card's PARAPHRASES correct  # the new wording
gap           = perf_accuracy − memory_recall
```

Averaged over the 29 cards. The verdict rule (from the script):

- `mean |gap| < 0.05` → **gap ≈ 0**: performance is mirroring memory. **Bridge NOT built.**
- `mean gap < −0.05` → **paraphrases harder than recall**: performance captures
  understanding beyond memorization. **Bridge exists.**
- otherwise → diverges; inspect per-card rows.

## What the responses are, and how they were derived

The analysis needs, per card, `memory_recall` (0–1) and `paraphrase_correct`
(a list of booleans). Those live in
[`content/responses.json`](../content/responses.json), generated by
[`content/scripts/gen_paraphrase_responses.py`](../content/scripts/gen_paraphrase_responses.py).

Both signals are **derived from the same Rasch / 1PL model the shipped app
uses** — a line-for-line port of the primitives in
[`content/tools/ablation.py`](../content/tools/ablation.py), themselves ported
from `anki/rslib/src/scheduler/adaptive.rs` (`sigmoid`, `difficulty_to_logit`
with `SCALE = 4.0`). Item difficulty for each card is read exactly the way
`adaptive.rs::note_difficulty` reads it: AI rating (`aidiff::` tag, else
`ai_difficulty.json`), else coarse `difficulty::easy|medium|hard` → 20/50/80,
then `b = (d/100 − 0.5) · SCALE`.

For a simulated learner with fixed true ability `θ` and an item at difficulty `b`:

- **Memory** — the student has drilled this *exact* card, so the familiar wording
  gives a retrieval advantage, modelled as an additive **familiarity bonus** on
  the ability axis:

  ```
  memory_recall = sigmoid((θ + FAMILIARITY_BONUS) − b)
  ```

- **Performance** — the paraphrase wording is *new*, so the familiarity bonus is
  gone; the student answers from true ability against the same difficulty. Each
  linked paraphrase is a Bernoulli draw:

  ```
  paraphrase_correct[i]  ~  Bernoulli( sigmoid(θ − b) )
  ```

**The one stated assumption is `FAMILIARITY_BONUS`** — the ability boost from
having memorized the exact card. It is fixed at **1.2 logits**. The gap is
whatever the model then produces against the deck's *real* difficulties; it is
**not tuned to a target**. Crucially, **if `FAMILIARITY_BONUS = 0` the two
signals coincide and the gap collapses to ≈ 0** — i.e. the exact failure mode
rubric 7d is probing for. A nonzero bonus is the honest claim that a memorized
card is easier than the idea behind it.

Deterministic parameters (fixed, in `gen_paraphrase_responses.py`):
`SEED = 7`, `THETA_TRUE = 0.6`, `FAMILIARITY_BONUS = 1.2`, `SCALE = 4.0`.
Re-running produces a byte-identical `responses.json`. Each row in
`responses.json` also carries a `_sim` block (the card's difficulty, `b`, `θ`,
bonus, and `p_paraphrase`) so a grader can see it was **derived, not hand-set**.

```

mean memory recall     : 0.868
mean paraphrase accuracy: 0.713
mean gap (perf - memory): -0.155   mean |gap|: 0.229
----------------------------------------------------

         understanding beyond memorization. Bridge exists.
```

Per-section rollup (same run):

| Section | Cards | Mean memory | Mean paraphrase | Gap |
|---|:---:|:---:|:---:|:---:|
| Word problems | 3 | 0.877 | 0.556 | **−0.322** |
| Critical Reasoning | 5 | 0.832 | 0.600 | **−0.232** |
| Reading Comprehension | 1 | 0.868 | 0.667 | **−0.201** |
| Arithmetic | 12 | 0.897 | 0.708 | **−0.189** |
| Algebra | 8 | 0.842 | 0.854 | **+0.012** |
| **All** | **29** | **0.868** | **0.713** | **−0.155** |

Reading the per-card rows honestly: a few cards with only 2–3 paraphrases show
extreme values (e.g. `Q-PS-013` and `V-CR-004` drew all-wrong on their
paraphrases; several easy Algebra cards drew all-right and land slightly
*positive*). That is genuine **small-sample Bernoulli noise** at 2–3 draws per
card — the Algebra section's tiny +0.012 is exactly that. It is **not** smoothed
away, because the aggregate over 29 cards × 83 questions is the number the rubric
asks for, and it is unambiguously negative (−15.5 pts).

## The honest caveat

- **No real students this week.** This shows the **mechanism**, not a measured
  human gap. Both signals come out of the app's Rasch/1PL model under one stated
  behavioural assumption (the familiarity bonus). It demonstrates that the
  performance model, *as built*, reads the new-wording items independently of the
  memorized card — it is **not** wired to copy the memory score.
- **The gap's sign is by construction; its size is not.** The familiarity bonus
  makes memory ≥ performance by design (that is the honest claim about drilled
  cards). But the *magnitude* (−15.5 pts) falls out of the model applied to the
  deck's real difficulties and the fixed seed — it was not dialled to a target.
- **What would make this a real result:** replay `responses.json` from actual
  student answers — `memory_recall` from each card's FSRS retrievability /
  pass-rate, and `paraphrase_correct` from the student answering the linked
  paraphrases in the app. The analysis script (`memory_vs_performance.py`) is
  already written to consume exactly that shape, so swapping the simulated
  `responses.json` for a real one needs **no code change**.

---

## Interpretation

A **nonzero, negative gap (−0.155)** means the performance model is **not just
echoing the memory model**: reworded questions that test the same idea are
measurably harder than recall of the memorized card. Memorizing a card ≠
understanding the idea — which is the whole reason the app reports Memory and
Performance as **separate** scores rather than one number. Had the gap come out
≈ 0, the honest conclusion would have been the opposite: performance is copying
memory, and the two scores are redundant.
